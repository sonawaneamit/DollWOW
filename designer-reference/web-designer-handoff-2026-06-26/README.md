# DollWow Web Design Handoff

Date: June 26, 2026

This packet is for a visual/web designer to polish the DollWow customer storefront without needing access to the full engineering repository.

## Recommended Workflow

Use the live storefront and the notes in this folder as the source of truth for design direction. Please create design recommendations, Figma frames, or standalone HTML/CSS references that preserve the current ecommerce flows. The engineering team will port approved ideas back into the Next.js/Shopify storefront.

Do not start from generic adult-site templates. DollWow should feel premium, discreet, emotionally warm, and very clear to buy from.

## What We Want Improved

- Make the storefront feel more designed and less text-heavy.
- Keep the Noir Boudoir direction: dark, intimate, refined, warm, premium.
- Add stronger visual hierarchy so shoppers can quickly understand product, customization, trust, and checkout steps.
- Improve homepage merchandising, PDP lower-page sections, cart, shop filters, and policy/trust pages.
- Make the website feel high-trust without fake reviews, fake buyer photos, or overexplaining.
- Keep copy simple and customer-facing. Avoid internal phrases like “path,” “flow,” “framework,” “data source,” “review layer,” or anything that sounds like product-management notes.

## Must Not Break

- Product image gallery and product thumbnails.
- Build/customization studio.
- Paid customization price updates.
- Add-to-cart/checkout behavior.
- Help Me Choose quiz.
- Price-match request form.
- Mobile sticky checkout behavior.
- Header navigation, footer policy access, and cart/search/help icons.
